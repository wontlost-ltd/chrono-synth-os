# v2.0.0-beta.1 — NAS Quick Start

**用途**：在 Synology NAS 上启动 chrono-synth-os + chrono-synth-web 的 v2.0.0-beta.1 内测部署
**前置**：你已经 `sudo docker pull ghcr.io/wontlost-ltd/chrono-synth-{os,web}:2.0.0-beta.1` 成功
**配套验收清单**：`internal-beta-checklist.md` NAS-01 ~ NAS-06

---

## 目录结构

整个内测部署只用 NAS 上一个目录，建议 `/volume2/docker/chrono-beta/`：

```
chrono-beta/
├── docker-compose.yml      ← 本指引第 3 步生成
├── .env                    ← 本指引第 2 步生成（含密钥，已 chmod 600）
├── Caddyfile               ← 本指引第 4 步生成
├── jwt-keys/
│   ├── kid-1.priv.pem      ← 第 2 步生成
│   └── kid-1.pub.pem       ← 第 2 步生成
└── data/                   ← 由容器自动创建（SQLCipher DB）
    ├── postgres/           （如果用 postgres 而非 sqlite）
    └── os/                 (chrono.db)
```

## 第 1 步：先把 ghcr.io 登录持久化（避免每次重启 pull 都要登）

如果你刚才 `docker login ghcr.io` 用 `--password-stdin` 喂了 PAT，DSM 重启后会丢。把它写进 docker 的 config（仍然安全，存在 `~/.docker/config.json`，0600 权限）：

```bash
ls -la ~/.docker/config.json
# 应该看到 -rw------- 1 rpang ... config.json
```

如果不存在，重跑：

```bash
echo '<你的 PAT>' | sudo docker login ghcr.io -u jet-pang --password-stdin
```

## 第 2 步：生成 JWT 密钥 + .env

```bash
cd /volume2/docker
mkdir -p chrono-beta/jwt-keys
cd chrono-beta

# 生成 RS256 密钥对
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 \
  -out jwt-keys/kid-1.priv.pem
openssl rsa -in jwt-keys/kid-1.priv.pem -pubout -out jwt-keys/kid-1.pub.pem
chmod 600 jwt-keys/kid-1.priv.pem

# 生成 .env（含所有运行时秘密 + 配置）
cat > .env <<'EOF'
# ─── 必填：基础设施 ───
POSTGRES_PASSWORD=__REPLACE__ChAnGe_Me__
# 32 字节 hex，用于 SQLCipher 字段加密
CHRONO_FIELD_ENC_MASTER_KEY=__REPLACE_64_hex_chars__

# ─── 必填：JWT 单 key 模式（beta 用）───
# 注：beta 用 legacy 单 key 模式。多 kid 热轮换 (jwt.keys ring) 在启动后
# 通过 /api/v1/auth/keys/rotate 动态添加，无需 config 预先声明所有 key。
# v2.0.0-beta.1 的 §8 #1 Critical fix 让 rotate 不再要求重启。
# 注：CHRONO_JWT_PRIVATE_KEY / PUBLIC_KEY 不支持 _FILE 后缀，必须用 PEM
# 字符串。下面 sed-PEM-into-env 的脚本把 PEM 的换行替换成 \n 文本字面量。
CHRONO_JWT_KID=kid-1
# CHRONO_JWT_PRIVATE_KEY 和 CHRONO_JWT_PUBLIC_KEY 由本指引第 2.5 步自动填入

# ─── 域名 ───
BETA_DOMAIN=chrono-beta.example.com   # 改成你 NAS 对外可达的域名
EOF

# 重要：把生成的密钥实际填进去
POSTGRES_PASS="$(openssl rand -hex 24)"
ENC_KEY="$(openssl rand -hex 32)"
sed -i "s|__REPLACE__ChAnGe_Me__|${POSTGRES_PASS}|" .env
sed -i "s|__REPLACE_64_hex_chars__|${ENC_KEY}|" .env

chmod 600 .env
```

### 第 2.5 步：把 PEM 字符串塞进 .env

OS 镜像的 `CHRONO_JWT_PRIVATE_KEY` env 收 PEM 字符串（不是文件路径），多行 PEM 必须把换行变成 `\n` 字面量，docker compose 启动时再还原。

```bash
# 把 PEM 多行变成单行 \n-分隔字符串，追加到 .env
PRIV_INLINE=$(awk '{printf "%s\\n", $0}' jwt-keys/kid-1.priv.pem)
PUB_INLINE=$(awk '{printf "%s\\n", $0}' jwt-keys/kid-1.pub.pem)

cat >> .env <<EOF

# JWT key material（PEM 字符串，换行已转 \n）
CHRONO_JWT_PRIVATE_KEY=${PRIV_INLINE}
CHRONO_JWT_PUBLIC_KEY=${PUB_INLINE}
EOF
```

验证 .env 没坏：
```bash
grep -c "^CHRONO_JWT_PRIVATE_KEY=" .env   # 期望 1
grep -c "^CHRONO_JWT_PUBLIC_KEY=" .env    # 期望 1
grep "BEGIN PRIVATE" .env | wc -l         # 期望 1（应在 CHRONO_JWT_PRIVATE_KEY 行）
```

> ⚠️ **替换 `BETA_DOMAIN`**：编辑 `.env`，把 `BETA_DOMAIN=` 那行改成你 NAS 上能解析到的域名（如果暂时没域名，写 NAS 内网 IP 也可，但 Caddy 的 HTTPS 证书会签不下来——见第 4 步 fallback）。

## 第 3 步：写 docker-compose.yml

```bash
cat > docker-compose.yml <<'YAML'
services:
  postgres:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: chrono
      POSTGRES_USER: chrono
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U chrono"]
      interval: 10s
      timeout: 5s
      retries: 5

  # 关键：service name MUST be 'backend' — chrono-synth-web 的 nginx.conf
  # 写死了 proxy_pass http://backend:3000，改名会导致 web 拉不到 API。
  backend:
    image: ghcr.io/wontlost-ltd/chrono-synth-os:2.0.0-beta.1
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      # 基础
      CHRONO_DB_DRIVER: postgres
      CHRONO_DB_CONNECTION_STRING: postgres://chrono:${POSTGRES_PASSWORD}@postgres:5432/chrono
      CHRONO_SERVER_HOST: 0.0.0.0
      CHRONO_SERVER_PORT: '3000'
      CHRONO_SERVER_PUBLIC_URL: https://${BETA_DOMAIN}
      CHRONO_LOG_LEVEL: info
      CHRONO_LOG_JSON: 'true'
      # JWT 单 key 模式（beta）
      CHRONO_JWT_ENABLED: 'true'
      CHRONO_JWT_ALGORITHM: RS256
      CHRONO_JWT_ISSUER: https://${BETA_DOMAIN}
      CHRONO_JWT_ACCESS_TTL_MS: '900000'         # 15 min
      CHRONO_JWT_REFRESH_TTL_MS: '2592000000'    # 30 day
      CHRONO_JWT_KID: kid-1
      # PEM 字符串从 .env 直读，docker compose 自动还原 \n 转义
      CHRONO_JWT_PRIVATE_KEY: ${CHRONO_JWT_PRIVATE_KEY}
      CHRONO_JWT_PUBLIC_KEY: ${CHRONO_JWT_PUBLIC_KEY}
      # Auth fallback (API key)
      CHRONO_AUTH_ENABLED: 'true'
      # CORS — web 同 host 走相对路径，正常不需要 origin allowlist
      CHRONO_CORS_ORIGIN: 'true'
      CHRONO_CORS_CREDENTIALS: 'true'
    volumes:
      - ./data/os:/app/data
    ports:
      - "127.0.0.1:3000:3000"   # 仅本地 + caddy 反代

  web:
    image: ghcr.io/wontlost-ltd/chrono-synth-web:2.0.0-beta.1
    restart: unless-stopped
    depends_on:
      - backend
    environment:
      CHRONO_WEB_API_BASE_URL: https://${BETA_DOMAIN}
      CHRONO_WEB_ENVIRONMENT: beta
    ports:
      - "127.0.0.1:8080:8080"   # 仅本地 + caddy 反代

  caddy:
    image: caddy:2-alpine
    restart: unless-stopped
    depends_on:
      - backend
      - web
    ports:
      - "443:443"
      - "80:80"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
      - ./data/caddy/data:/data
      - ./data/caddy/config:/config
    environment:
      BETA_DOMAIN: ${BETA_DOMAIN}

YAML
```

> ⚠️ **重要**：service name 必须是 `backend`（不是 `os` 或 `chrono-os`）。chrono-synth-web 的 nginx.conf 把所有 `/api/*` proxy 到 `http://backend:3000`，改名 web 就拉不到 API。

## 第 4 步：Caddyfile

```bash
cat > Caddyfile <<'CADDY'
{$BETA_DOMAIN} {
    encode gzip

    # SSE 长连接：禁用 buffering + 拉长超时（feature-flag kill-switch 依赖）
    handle /api/v1/feature-flags/stream {
        reverse_proxy backend:3000 {
            transport http {
                read_timeout 24h
            }
            flush_interval -1
        }
    }

    # JWKS 端点必须可外网访问（用于第三方验证 JWT）
    handle /.well-known/jwks.json {
        reverse_proxy backend:3000
    }

    # 其它 API 一律走 backend
    handle /api/* {
        reverse_proxy backend:3000
    }

    # 健康检查
    handle /healthz {
        reverse_proxy backend:3000
    }
    handle /readyz {
        reverse_proxy backend:3000
    }

    # 默认走 web（nginx 静态 + 自己处理 /api/* 内部 proxy；
    # 但外层 caddy 已经截走 /api/*，到这里只剩 SPA 资源）
    handle {
        reverse_proxy web:8080
    }
}
CADDY
```

### 域名 fallback：如果你只用 NAS 内网 IP

如果暂时没有可解析的域名，把 Caddyfile 改成：

```caddy
:443 {
    tls internal      # 用 Caddy 自签证书，浏览器会警告但能用
    # ... 其余 handle 保持不变
}
:80 {
    redir https://{host}{uri} permanent
}
```

并把 `.env` 的 `BETA_DOMAIN` 改成 `<NAS-IP>` 例如 `192.168.1.10`。

### NAS-03 JWT 热轮换验证（v2.0.0-beta.1 的 Critical fix）

```bash
# Step 1: 通过 register API 创建用户（密码用 argon2 哈希，直接走 API
# 而不是 raw SQL —— SQL 插入需要预先算 argon2 hash，太麻烦）。
curl -sk -X POST https://${BETA_DOMAIN}/api/v1/auth/register \
  -H 'content-type: application/json' \
  -d '{"email":"admin@beta.test","password":"password123"}'
# 期望：{"data":{"userId":"user_...","accessToken":"...","refreshToken":"..."}}

# Step 2: 把该用户升级到 admin 角色（admin 才能调 rotate API）
sudo docker compose exec postgres psql -U chrono -c "
  UPDATE users SET role='admin' WHERE email='admin@beta.test';
"

# Step 3: 重新登录拿带 admin role 的 token
TOKEN=$(curl -sk -X POST https://${BETA_DOMAIN}/api/v1/auth/login \
  -H 'content-type: application/json' \
  -d '{"email":"admin@beta.test","password":"password123"}' \
  | jq -r '.data.accessToken')
echo "TOKEN=${TOKEN:0:30}..."

# 解码 token header，应看到 kid=kid-1
echo "$TOKEN" | cut -d. -f1 | base64 -d 2>/dev/null | jq

# 生成第二把 kid-2 用于轮换
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 -out /tmp/kid-2.priv.pem
openssl rsa -in /tmp/kid-2.priv.pem -pubout -out /tmp/kid-2.pub.pem
PRIV2=$(awk '{printf "%s\\n", $0}' /tmp/kid-2.priv.pem)
PUB2=$(awk '{printf "%s\\n", $0}' /tmp/kid-2.pub.pem)

# 轮换：v2.0.0-beta.1 不再返回 409 AUTH_ROTATE_RESTART_REQUIRED
ROTATE=$(curl -sk -X POST https://${BETA_DOMAIN}/api/v1/auth/keys/rotate \
  -H "authorization: Bearer ${TOKEN}" \
  -H 'content-type: application/json' \
  -d "{
    \"newActiveKid\":\"kid-2\",
    \"addNew\":[{
      \"kid\":\"kid-2\",\"state\":\"grace\",\"algorithm\":\"RS256\",
      \"privateKey\":\"${PRIV2}\",
      \"publicKey\":\"${PUB2}\",
      \"secret\":\"\"
    }]
  }")
echo "$ROTATE" | jq
# 期望：{"ok":true,"activeKid":"kid-2","snapshot":...}（HTTP 200，不是 409）

# 重新登录，新 token 的 kid 应该是 kid-2
TOKEN2=$(curl -sk -X POST https://${BETA_DOMAIN}/api/v1/auth/login \
  -H 'content-type: application/json' \
  -d '{"email":"admin@beta.test","password":"password123"}' \
  | jq -r '.data.accessToken')
echo "$TOKEN2" | cut -d. -f1 | base64 -d 2>/dev/null | jq
# 期望 header.kid="kid-2"
```

如果上面 ROTATE 返回 `{"ok":true,"activeKid":"kid-2"}` 且第二次登录 token 的 `header.kid` 已经是 `kid-2`——**§8 #1 Critical 修复在真实部署里生效**，本次 beta 最关键的验证点通过。

## 常见问题

### `web` 容器拉不到 API（502 / "无法连接"）

检查 docker-compose.yml 里 OS service 名字是否是 `backend`。nginx.conf 写死 `proxy_pass http://backend:3000`，service 改名 → 内部解析失败。

### Caddy 申请 HTTPS 证书失败

- 域名 DNS 没指向 NAS 公网 IP
- NAS 80 端口没暴露到公网（Let's Encrypt 要走 HTTP-01 challenge）
- 改用 `tls internal` 走 Caddy 自签（见第 4 步 fallback）

### `/api/v1/auth/login` 报 500

容器内 postgres 没 ready 或 schema 没初始化。查日志：

```bash
sudo docker compose logs backend | grep -iE "(error|migration)"
```

期望看到迁移完成日志：`dsl-migrations-runner: applied N migrations`。如果迁移失败，最常见是 `CHRONO_DB_CONNECTION_STRING` 拼错。

### JWT 私钥读不到

```bash
sudo docker compose exec backend ls -la /run/secrets/
# 期望看到 jwt_priv.pem + jwt_pub.pem 都存在，且权限 0444（容器内 secret 默认权限）
```

如果空 → docker-compose.yml 的 `secrets:` 节没写对，或 `./jwt-keys/kid-1.priv.pem` 路径不对。

### Cosign 验证 image 签名（可选 — bonus 安全性验证）

```bash
# 在 NAS 上装 cosign
wget -O cosign https://github.com/sigstore/cosign/releases/latest/download/cosign-linux-amd64
chmod +x cosign

# 验证 image 签名（keyless OIDC，证书 identity 必须是 release.yml）
COSIGN_EXPERIMENTAL=1 ./cosign verify \
  --certificate-identity-regexp='https://github\.com/wontlost-ltd/chrono-synth-os/\.github/workflows/release\.yml@.*' \
  --certificate-oidc-issuer=https://token.actions.githubusercontent.com \
  ghcr.io/wontlost-ltd/chrono-synth-os:2.0.0-beta.1
```

期望输出：`Verification for ghcr.io/wontlost-ltd/chrono-synth-os:2.0.0-beta.1 -- The following checks were performed: ...`
